The first example demonstrates that subtracting an xs:date from a node casts the node to an xs:double, despite the fact that it's always illegal to subtract a date from a number.

The second example demonstrates that the result of any arithmetic that involves a node is an xs:double, which isn't a legal type for functions and operators that require xs:integer, such as the 'to' operator. Whereas "1 to @length" is absolutely fine, "1 to (@length + 1)" is not because "@length + 1" returns an xs:double, which can't be cast automatically to an xs:integer.
