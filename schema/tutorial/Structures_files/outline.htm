Content Model
"All" Content Models
xs:all says "all of these elements, in any order"
Common requirement in data-oriented XML
Some pretty severe limitations:
Can only contain element particles
Can't say "this sequence and this choice in any order"
Elements can only be optional or appear just once
Can't say "element X and five of element Y, in any order"
Cannot appear within other content models
Can't have a choice between two all content models
Cannot occur more than once
 maxOccurs for xs:all is always 1
"All" content models
Element Wildcards
Use xs:any in place of xs:element for "any element"
Limit namespaces with namespace attribute
##all (default)  any namespace
##other  any namespace other than the target namespace
List of namespace URIs
##local for no namespace
##targetNamespace for target namespace
Element Wildcards
Validating Wildcards
Wildcard validation controlled by processContents attribute
strict (default)  invalid if no element declaration
lax  doesn't validate if there isn't an element declaration
skip  always valid (never validates)
Use wildcards to provide extensible schemas
Mixed Content
Mixed content indicated with mixed attribute on xs:complexType with the value true
Mixed content can coexist with a normal content model
Possible to constrain order and occurrence of elements in mixed content
Determinism
Like DTD content models, content models in XML Schema must be deterministic
At any point, a validator must know which particle to use to validate a particular element
Validators use only the QName of an element to determine the particle, not its type
Can't say "a choice between an X element of type A or an X element of type B"
Somewhat complicated by
Wildcards make it easier to accidentally create non-deterministic content models
Numerical occurrence indicators make ambiguous content models possible

Global and Local Declarations
Local Declarations
Global declarations at top level of schema
Only one allowed for a particular name
Local declarations within content models
Doesn't override existing declaration (as in XDR)
Cannot be referred to from other model groups
Local Declarations and Namespaces
Global declarations declare elements in the target namespace of the schema
Local declarations can declare elements in the target namespace of the schema or in no namespace
Depends on the form attribute on the declaration or the elementFormDefault or attributeFormDefault attribute on xs:schema
qualified means in target namespace
unqualified (default) means in no namespace

Default and Fixed Values
Specify default value with default attribute
Apply if attribute is missing
Apply if element is present but empty
Specify fixed value with fixed attribute
Values must be valid against type
Can't specify default or fixed value for elements or attributes of xs:ID type

Summary
Content models similar to DTD but:
Can specify "all" model groups
Can use wildcards in place of named elements
Can constrain element occurrence in mixed content
Elements and attributes can be declared globally or locally
Watch the namespaces!
Elements can have fixed or default values
Model groups and groups of attributes can be named and referred to for reusability
