Determinism
•Like DTD content models, content models in XML Schema must be deterministic
•At any point, a validator must know which particle to use to validate a particular element
•Validators use only the QName of an element to determine the particle, not its type
•Can't say "a choice between an X element of type A or an X element of type B"
•Somewhat complicated by
•Wildcards make it easier to accidentally create non-deterministic content models
•Numerical occurrence indicators make ambiguous content models possible
