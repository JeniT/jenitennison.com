What is a Data Type?
Data types are defined by their value space
Set of values that are valid for that data type
E.g. 42 is a valid decimal value
Each value can have one or more lexical representations
Actual text present in XML document
E.g. 42 could be represented as 42 or 42.00 or 00042 or …
One lexical representation is the canonical lexical representation
E.g. 42.0
Validity and Whitespace
Values of a particular type must be in one of the valid lexical representations for that type
For most data types, whitespace is collapsed prior to assessment
Whitespace characters replaced by spaces
Leading and trailing whitespace stripped
Runs of spaces replaced by single space
Exceptions are:
 xs:string – whitespace unaffected
 xs:normalizedString – whitespace replaced

All user-defined data types are derived from built-in types

Dates and Times
ISO 8601 formats
Dates and times
xs:dateTime - CCYY-MM-DDThh:mm:ss
xs:date - CCYY-MM-DD
xs:time - hh:mm:ss
Gregorian dates:
xs:gYear – CCYY
xs:gYearMonth – CCYY-MM
xs:gMonth – --MM
xs:gMonthDay – --MM-DD
xs:gDay – ---DD
All can have timezones – Z or (+|-)hh:mm
Dates and Times
Durations
Use xs:duration data type
Format is (-)PnYnMnDTnHnMnS
Specifies years, months, days, hours, minutes, seconds
Any component can be missing
E.g. P1Y1D, PT2S, P14D, P9M
Two important subcategories
Year/month durations
(-)PnYnM
Day/time durations
(-)PnDTnHnMnS

Defining New Simple Types
Use xs:simpleType
Anonymously, nested within xs:element or xs:attribute
With a name attribute at the top level of the stylesheet
New simple type belongs to target namespace of schema
Data Type Classification
Atomic types
Single values e.g. "a date"
List types
Whitespace-separated lists of values
No way to specify a different separator
Can't have lists of lists, or lists of values that can contain whitespace (e.g. strings)
Union types
Combinations of data types e.g. "an integer or the token 'Infinity'"
Can have unions of list types, lists of union types

Facets
Most user-defined types restrict values through facets
Lengths
xs:length, xs:minLength, xs:maxLength
Ranges
xs:minExclusive, xs:minInclusive, xs:maxInclusive, xs:maxExclusive
Precision and scale
xs:totalDigits, xs:fractionDigits
Can restrict lexical representation with xs:pattern
This doesn't change the valid lexical representations
Length
Units of length depend on data type
Strings and URIs use characters
Binary data types use bytes
List types use number of items in list
Ranges
Ranges can apply to numbers, to date/times and to durations
Digits
Can specify maximum digits allowed in total or after the decimal point
This doesn't fix the number of digits in the lexical representation, just in the value
Patterns
Patterns are the only facet that limit the lexical representation
Use as a last resort – other facets are a lot easier for applications to use because they can act as a guide rather than just a test
Patterns use Perl-based regular expression syntax
Always match the entire string
Can match Unicode characters
Multiple xs:pattern elements provide alternative regular expressions
